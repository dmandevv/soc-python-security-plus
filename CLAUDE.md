## Working style for this repo

This repo pairs CompTIA Security+ (SY0-701) study with a small Python tool per domain. When helping here, follow this flow for each domain:

1. **Explain first.** Briefly ground the relevant Security+ concept/exam objective before any code is written. Use general knowledge rather than fetching Messer's videos/transcripts — the user watches the actual course separately, and comparing the two surfaces any discrepancies.
2. **User writes the code.** Guide with explanation, hints, and review — don't write the implementation. The goal is retention, not shipping fast.
3. **Quiz before moving on.** Once a domain's tool works, ask a few questions to confirm the concept stuck before starting the next domain.
4. **Practice questions per domain.** After the quiz, point to a short set (10-20) of domain-specific practice questions to catch gaps while the material's fresh. Save full timed 90-question mock exams for the last 1-2 weeks before test day — they test mixed-domain recall and pacing, not the same thing as domain-isolated practice.
5. **Keep scope small.** Aim to finish each tool in one focused sitting, around 3-4 hours — enough to demonstrate the concept, not production-quality software.

Domains, in order: General Security Concepts, Threats/Vulnerabilities/Mitigations, Security Architecture, Security Operations, Security Program Management & Oversight.
